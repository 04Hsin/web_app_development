# PRD：食譜收藏夾

## 1. 專案概述

### 背景與動機
現代人生活繁忙又注重健康，許多人有自煮的需求，但在網路上看到喜歡的食譜後，往往散落在各大社群平台或瀏覽器書籤中，難以統整與回顧；此外，打開冰箱不知道剩下的食材能做什麼料理也是一大痛點。因此，本系統希望能提供一個專屬的「食譜收藏夾」，讓使用者能統一管理並智慧搜尋食譜。

### 目標用戶
- **一般用戶（食譜愛好者、自煮族群）**：需要收藏、管理與搜尋食譜的使用者。
- **管理員（Admin）**：負責維護系統平台、管理使用者名單與整體食譜庫的營運人員。

### 核心價值主張
讓儲存與尋找食譜變得簡單直覺。不僅能有系統地建立個人食譜庫，更獨創「食材組合搜尋」功能，解決冰箱剩食不知道如何料理的困擾。

---

## 2. 功能需求

### 功能清單
1. **儲存/新增食譜**
2. **瀏覽與搜尋食譜**
3. **食材組合搜尋（冰箱尋寶）**
4. **使用者註冊與登入（會員系統）**
5. **管理員後台管理**

### 使用者故事 (User Stories)

**1. 儲存/新增食譜**
- 作為 **一般用戶**，我希望 **能手動建立或儲存網路上看到的食譜（包含名稱、食材清單、步驟、圖片等）**，以便 **我日後要煮飯時能隨時查看**。

**2. 瀏覽與搜尋食譜**
- 作為 **一般用戶**，我希望 **能透過關鍵字（如：咖哩、雞肉）搜尋已收藏的食譜**，以便 **我能快速找到我想做的特定料理**。
- 作為 **一般用戶**，我希望 **能依據分類（如：早餐、甜點、晚餐）瀏覽食譜**，以便 **在沒有明確想法時獲得料理靈感**。

**3. 食材組合搜尋（冰箱尋寶）**
- 作為 **一般用戶**，我希望 **能選擇或輸入目前擁有的多項食材（如：雞蛋、番茄）進行搜尋**，以便 **系統能推薦我可以使用這些食材製作的對應食譜，解決剩食困擾**。

**4. 使用者註冊與登入（會員系統）**
- 作為 **一般用戶**，我希望 **能註冊專屬的個人帳號並登入**，以便 **我的食譜收藏不會與其他人混淆，且更換設備也能存取資料**。

**5. 管理員後台管理**
- 作為 **管理員**，我希望 **能查看系統數據與管理（刪除/封鎖）違規的使用者與食譜**，以便 **確保平台內容的品質與正常營運**。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架
- **前端渲染**：使用 Jinja2 模板引擎與原生 HTML/CSS/JS (不依賴複雜的單頁應用框架)
- **資料庫**：使用 SQLite 作為輕量級關聯式資料庫

### 效能與安全考量
- **響應式設計 (RWD)**：考量使用者常在廚房邊看手機邊料理，介面需在手機與平板上有良好的顯示效果。
- **安全性 (Security)**：密碼需經過 Hash 雜湊處理後儲存，並防範基本的 SQL Injection 與 XSS 攻擊。
- **效能 (Performance)**：食材聯集/交集搜尋的資料庫 Query 需要優化，避免當食譜數量多時搜尋速度過慢；圖片上傳應有大小限制。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 使用者註冊、登入與登出功能。
- 專屬個人的食譜CRUD（新增、瀏覽、編輯、刪除）。
- 標題與內文關鍵字的基本食譜搜尋。
- 基於單一或多個食材標籤進行食譜搜尋的基礎功能。

### Should Have (應該具備)
- 食譜的圖片上傳與顯示。
- 食譜分類標籤（如：台式、西式、甜點）。
- 簡易的管理員後台（可查看系統所有的食譜與使用者列表）。

### Nice to Have (最好具備)
- 從外部網址自動擷取（爬蟲）食譜內容儲存。
- 收藏/追蹤「其他使用者」公開的食譜。
- 給食譜打分數或留言評論。
